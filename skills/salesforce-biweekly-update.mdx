<Info title="Skill detail">
Every two weeks, each CSM writes a status update for every account in their book and pastes it into Salesforce. This skill does the heavy lifting: it pulls from your Granola meeting notes, Slack threads, and Gmail from the last 1-2 weeks, then generates an executive-ready, plain-text update formatted for direct copy-paste into Salesforce. No markdown, no formatting issues, no starting from a blank text box.
</Info>

Ready to install? Head to the [Install & available skills](/skills/install) page to download.

## Prerequisites

Before using this skill, make sure you have:

- **Code execution enabled** — Settings → Capabilities → Code execution and file creation
- **Connectors enabled** — The skill pulls context automatically from three sources. Go to Customize → Connectors and make sure these are connected:
  - **Granola** — Your meeting transcripts. This is the richest data source. If you're not running Granola on your customer calls, the skill will still work but the output won't be as strong.
  - **Slack** — Messages and threads mentioning your accounts across customer channels, internal CS channels, and engineering/product channels.
  - **Gmail** — Email threads with or about your accounts.

## How to use

### Option 1: Cowork automation (recommended)

This is the intended workflow. Set up a recurring Cowork task that runs every other Friday and generates updates for all your accounts automatically.

**Step 1:** Run `/salesforce-update` for the first time. Claude will ask you to fill out a template with your account details:

```
Account: [Account Name]
Slack channels: [comma-separated channel names]
Email domains: [comma-separated @domain.com patterns for this customer]
Workstreams (optional): [key project names or topics to search for]
```

Fill this out for each account you manage. Claude saves it to memory automatically — you only do this once.

**Step 2:** Set up the scheduled task. In Cowork, use `/schedule` and configure:

- **Frequency:** Every other Friday (align to your team's bi-weekly cadence)
- **Prompt:** "Using the salesforce-biweekly-update skill, generate a Salesforce bi-weekly update for each of my accounts. Search Granola, Slack, and Gmail for context from the last 7-14 days. Save each update as a separate .txt file."
- **Connectors:** Granola, Slack, Gmail

**Step 3:** Review, edit, and paste. Cowork generates a .txt file for each account. Open each one, review the content, make any edits, then paste into Salesforce. The output is plain text so it pastes cleanly with no formatting issues.

### Option 2: Ask Claude in chat

If you need a quick update or want to try the skill before setting up Cowork, open a new conversation and say:

```
Write my bi-weekly update for Accelerant
```

Claude will search your Granola meetings, Slack, and Gmail for the last 7-14 days of activity on that account and generate the update. You can also do multiple accounts at once:

```
Write my bi-weekly updates for Ascot, Kettle, and Velocity Risk
```

### Option 3: Paste your own notes

If you don't have connectors set up or want to provide specific context:

```
Here are my notes from this week. Write my bi-weekly update for UFG:

[paste your notes here]
```

## What the output looks like

The output is plain text formatted for Salesforce. ALL CAPS headers with a dash, then 3-5 sentences of narrative per section with specific names, dates, and numbers. Active implementation accounts will typically have 4-8 sections. Steady-state accounts might have 2-3.

```
03/04/26

UAT ENVIRONMENT LAUNCHED - Jackson delivered UAT credentials to the
full Accelerant team on 2/26 including Jeffrey Noe, Alex Carr, Reese
Mularz, Thomas Zack, Cathleen Gawlik, and Venkat Raman. A password
reset bug was identified same-day and patched that evening. PACE team
received separate credentials for external API connections on 2/25,
with updated vault links sent 3/3 expiring 3/13.

PREMIUM FINANCING - Made the decision to move from APFC/Vertafore to
IPFS for premium financing. Vertafore's timeline was not going to get
us to launch, and going live without a financing option would
negatively impact hit ratios. Alex has communicated this change to
the Leavitt team. We are not closing the door on APFC permanently -
once Vertafore is ready to deliver a great experience we are happy to
revisit.

GO-LIVE TIMELINE - Binding target is now May 1 with earliest policy
effective date of May 1, moved from the original March 1 target. The
delay is driven by forms complexity and IPFS integration timeline.
Estimated missed revenue from the April delay is approximately $2M
against a 2026 target of $20M booked / $80M run rate.

BILLING - Had a productive session with Goldman Sachs to
pressure-test the platform fee structure. Landing on a 1% platform
fee for ACH with a higher rate for credit cards, working through cap
structure for larger premiums. Core billing functionality is
feature-complete and ready for UAT.
```

## Tips for getting the best output

- **Run Granola on your customer calls.** Granola meeting notes are by far the richest data source. If Granola doesn't have your meetings, the skill is working with much less context.
- **The skill auto-expands the search window.** It starts with the last 7 days. If there aren't enough substantive topics, it automatically expands to 14 days and notes that at the top of the output.
- **You can edit after the fact.** If the output is missing something, tell Claude: "Add a section about the billing discussion from Tuesday's call." If something is wrong: "The go-live date should be April 15, not May 1 — fix that section."
- **Low-activity accounts get shorter updates.** The skill generates 2-3 sections for quiet accounts rather than padding with filler. That's intentional.
- **To update your accounts**, just tell Claude: "Add a new account," "Update my Accelerant channels," or "Remove UFG from my accounts." No file editing needed — it updates memory automatically.

## What this skill does NOT do

- It does not post directly to Salesforce — the output is for copy-paste
- It does not set account health status (Red/Yellow/Green) — you add that yourself
- It does not create action items or assign owners — it reports what happened and what's coming
