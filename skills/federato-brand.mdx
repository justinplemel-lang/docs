## Slide generation skill (brand CSS)

A Claude skill for creating **on-brand Federato presentations**. It defines colors, typography, slide layouts, and design patterns so every deck stays consistent with the brand. The skill works with the base pptx workflow and includes pre-build discovery, compact helpers for large decks, and reference files.

Ready to install? Head to the [Install & available skills](/skills/install) page to download.

## What's in the updated skill

### Core instruction (SKILL.md)

- **Pre-build discovery** — Asks for deck type, audience, slide count, agenda, “walk out with” outcomes, and source material (e.g. Granola transcripts, prior decks) before building.
- **Large deck construction (15+ slides)** — Compact helper functions (footer, crosshair, open-question bar, photo section divider), data-driven slide loops, and guidance to stay within output limits.
- **Brand foundation** — The Pathfinder symbol usage and brand voice (confident, technical-but-accessible, forward-looking, data-driven, discovery-oriented).
- **Color palette** — Primary (Meridian Maroon, Compass Cream, Atlas Green, Signal Orange, Beacon Teal), secondary (Grounded Brown, Insight Purple, Lumen Yellow), background colors, and usage rules. No pure black; Meridian for anchors; Grounded Brown for cards and table headers.
- **Typography** — Frank Ruhl Libre (display/headlines), Inter (body). Explicit pptxgenjs usage: every `addText()` uses one of these. CSS variables for html-to-pptx.
- **Crosshair grid system** — Signal Orange vertical/horizontal lines with diamond connector; placement rules; when to use (content slides, agendas) and when not (Walk Out With, centered cards). Photo section dividers use crosshair in Compass Cream on top of the overlay.
- **20+ slide layouts** — Title (dark, halftone), Agenda, Section divider (solid + nature photography), Content (grid), Two-column comparison (ownership), Two-column halftone (title + content), Metrics/stats, Timeline (milestone + GTM arrow), Dependency/blockers table, Walk Out With, Discovery questions, Scoring matrix, Implementation scenario 2x2, UAT/roles table, Flywheel/cycle, GTM three-column, Current state (working/in progress), Cover/closing, Day recap, Success criteria, Capacity / “the math.”
- **Footer format** — Standard external (logo, CONFIDENTIAL, page number) and internal/partner variants.
- **Visual elements** — Halftone dots, Pathfinder usage, open-question callout bar, numbered badges, flywheel stage pills.
- **Content patterns** — “Old Core vs. Federato,” ownership split, discovery questions, metrics callout, table structure.
- **Checklist before finalizing** — Pre-flight QA so decks meet brand and structure rules.
